sort the flat_map. So long as you only manipulate the content of the map using it's methods, you do not need to reorder it.

for the block tensor class, some manipulation of the index could be necessary. If the transformation on the index changes the ordering, call sort afterward. Or face the consequences. All the method of this class are UB if it is not sorted.

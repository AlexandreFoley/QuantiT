\doxysubsection{Detailed Description}
polymorphic (type-\/erased?) container of \mbox{\hyperlink{classquantit_1_1any__quantity}{any\+\_\+quantity}}. All the element in the container are of the same concrete type, but the concrete type doesn\textquotesingle{}t change the type of the container. Implement most of the interface of std\+::vector (some parts are templates that can\textquotesingle{}t be replaced with run time polymorphism). 

While iterating on the polymorphic container is possible, and useful for prototyping, doing so will incur a significant cost due to virtual calls. {\bfseries{For perfomance critical operations on the whole collection, it is necessary to implement the loop as polymorphic function that will resolve in O(1) polymorphic call/dynamic\+\_\+cast.}}

The visitor pattern could also be used to accomplish efficient iteration on the vector, but a visitor cannot be made to work with all the types in an open set of types.

Note\+: I can\textquotesingle{}t help but feel there\textquotesingle{}s one too many inderection level necessary here (pointer to conc\+\_\+vect.\+data, data is a pointer to the actual data) There\textquotesingle{}s almost certainly a design without this extra indirection and the same interface.
